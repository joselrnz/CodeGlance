# Glance Walkthrough

Glance is the interactive `glance.html` output generated by CodeGlance. It is meant to serve two
audiences from the same analysis pass:

- Humans get a visual map, inspector, guided tour, source snippets, wiki, and local output browser.
- Agents get a compact read order, schema, TOON graph, JSON graph, and refresh rules so they can
  understand a repo without reading every file first.

## Quick Run

Generate every artifact:

```bash
python -m codeglance generate . --out .codeglance/outputs --profile all
```

Serve the output folder on the current machine:

```bash
python -m codeglance serve .codeglance/outputs --host 0.0.0.0 --port 8777
```

If you only need a basic static server:

```bash
python -m http.server 8777 --bind 0.0.0.0 --directory .codeglance/outputs
```

Open the local app:

```text
http://127.0.0.1:8777/glance.html
```

Open it from another device on the same Wi-Fi by using the machine LAN IP:

```text
http://<your-lan-ip>:8777/glance.html
```

## Generated Files

| File | Audience | What it is | When to use it |
| --- | --- | --- | --- |
| `index.html` | Human | Output launcher for the generated folder. | Start here when browsing all generated artifacts. |
| `glance.html` | Human | Interactive visual codebase map. | Explore layers, files, classes, functions, source snippets, and tours. |
| `wiki.html` | Human | Readable generated wiki. | Use when you want documentation instead of a graph. |
| `graph.static.html` | Human | Static graph view with no interactive canvas dependency. | Use for simple sharing or low-interaction review. |
| `context.md` | Human/agent | Full Markdown context. | Use for deep codebase reading, reports, or handoff docs. |
| `agent.md` | Agent | Compact low-token repo handoff. | First file an AI agent should read after `llms.txt`. |
| `onboarding.md` | Human/agent | First-day guide with read-first files, layers, tour order, and workflow. | Use when joining a repo or handing it to another agent. |
| `impact.md` | Human/agent | Changed-file impact report with dependency ripples and review checklist. | Use before commit or code review. |
| `llms.txt` | Agent | Tiny entrypoint and artifact read order. | Tell an agent where to start and what to avoid reading too early. |
| `llm-context.schema.json` | Agent/tool | Schema contract for generated artifacts. | Validate parsers and understand fields before consuming JSON/TOON. |
| `knowledge-graph.toon` | Agent | Compact graph table format. | Use for prompt context when JSON is too expensive. |
| `knowledge-graph.json` | Tool/agent | Canonical graph data. | Use for custom renderers, indexing, or precise graph queries. |
| `meta.json` | Human/tool | Analysis metadata. | Check version, analyzed files, node count, and generation state. |

## Glance Interface

### Top Bar

The top bar gives the shortest path into the repo:

- `Overview`: default landing view for normal users. Shows one card per layer so the project feels understandable first.
- `Drill`: file/class view. Use this when the full graph is too busy but you need to inspect real code structure.
- `Explore`: engineer view. Shows the richer graph, including functions and lower-level symbols.
- `Tour`: guided walkthrough through the important areas.
- `Structural`: dependency and containment graph.
- `Domain`: grouped inferred business/domain areas when detected.
- `Knowledge`: docs and Markdown knowledge graph.
- Search: finds nodes by name, path, summary, and related text.

On mobile, the top mode row is intentionally compacted. The same controls move into the Tools sidebar.

### Left Tools Sidebar

Desktop opens with the Tools sidebar visible. Mobile opens with a compact `Tools` rail to avoid covering
the graph.

The sidebar contains:

- View controls: Overview, Drill, Explore, Tour.
- Map controls: Structural, Domain, Knowledge.
- Analysis controls: Diff, Functions, Facets.
- Detail controls: Files and Classes.
- Search controls: Fuzzy and Semantic.
- Actions: Fit, Path, Filter, Export, Flow, Theme, Terminal, Help.

Use this sidebar for stable navigation. It replaces the old crowded top-right "More" menu.

### Right Inspector

Desktop opens with the Inspector visible. Mobile starts collapsed and exposes a bottom-right Inspector
pill when needed.

The Inspector shows:

- selected file/class/function metadata
- source path
- summary
- tags
- layer
- connections
- source snippet
- file tree tab

This is the main human review surface after clicking a node.

### Bottom Terminal

The terminal opens from the `>_` button at the bottom-left or from Tools -> Terminal.

It is offline and works inside the generated HTML. It is currently for guidance and graph commands,
not for running shell commands on the host machine. That keeps the artifact safe to open locally or
from a phone.

Useful direction for later:

- add repo-specific help commands
- add guided "what should I inspect next?" prompts
- add human approval checkpoints
- add current-folder command recipes that print instructions instead of executing anything

### Zoom Controls

Zoom controls sit bottom-center. They stay out of the Tools sidebar, Inspector, Tour card, and Terminal.
When the terminal opens, the zoom controls lift above it.

## Recommended Human Workflow

1. Generate all outputs with `--profile all`.
2. Open `index.html` or the served local URL.
3. Start in `Overview` to understand the main layers.
4. Open `Tour` for the guided path.
5. Use `Drill` when you want files/classes without function noise.
6. Use `Explore` when you need the full engineer-level graph.
7. Click nodes and read the Inspector source snippets.
8. Use the Terminal for local guidance and graph-oriented help.
9. Run `codeglance explain <path-or-symbol>` when a specific file or symbol needs a focused explanation.
10. Run `codeglance impact` before committing.
11. Change code, regenerate outputs, and compare the updated graph.

## Recommended Agent Workflow

Agents should read generated context in this order:

1. `llms.txt`
2. `agent.md`
3. `onboarding.md` when the task needs first-day orientation
4. `impact.md` when the task involves edits or review
5. `llm-context.schema.json`
6. `knowledge-graph.toon`
7. `knowledge-graph.json` only when exact graph data is needed
8. source files only after the generated map identifies the relevant area

This keeps token usage low. The agent starts from a summary, follows the graph to the right files, then
opens exact source only when a task requires it.

After edits, regenerate:

```bash
python -m codeglance generate . --out .codeglance/outputs --profile all
```

Then the agent should re-read `llms.txt`, `agent.md`, and the changed graph sections before continuing.

## Human-In-The-Loop Pattern

Use Glance as a review loop:

1. Agent generates or updates code.
2. CodeGlance regenerates outputs.
3. Human opens `glance.html` and checks Overview, Drill, Inspector, and Wiki.
4. Human flags confusing areas or missing docs.
5. Agent reads the compact context and updates only the relevant code/docs.
6. Repeat until the graph, wiki, and source agree.

This gives the human a visual checkpoint and gives the agent a compact memory checkpoint.

## Mobile Expectations

Validated mobile behavior:

- top mode row is hidden on phone-sized screens
- Tools rail opens the full tools drawer
- Inspector is available as a bottom-right pill
- terminal opens as a bottom dock
- zoom controls remain bottom-center and move above overlays
- Tour remains usable without removing the app chrome
- no targeted overlaps between top bar, tools, inspector, terminal, tour, and zoom controls
